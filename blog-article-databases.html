Local databases in Flutter: SQLite, Hive, or Drift?

A practical comparison based on real production use — covering performance, query flexibility, and developer experience for offline-first apps.

WA

Waseem Akram

Lead Mobile Developer & Software Engineer

Offline-first is a requirement in most of the apps I build — healthcare, logistics, enterprise HRMS. When the network is unreliable, the app still needs to work. Choosing the right local storage solution is a decision that compounds over the life of the project.

sqflite — the raw SQLite option

sqflite gives you direct access to SQLite with no abstraction layer. You write SQL strings, execute them, and map results manually. It's powerful and flexible, but the developer experience is rough. No type safety, no query builder, raw strings everywhere — mistakes at runtime that a compiler could have caught.

I use sqflite when I need maximum control over a complex schema or when I'm migrating an existing SQLite database from a native Android app. For greenfield projects, there are better options.

Hive — fast NoSQL for simple data

Hive is a key-value store written in pure Dart. It's extremely fast, requires no native dependencies, and works on every Flutter platform including Web. The API is simple and the performance on reads is excellent.

The limitation is its document model. Hive works well for storing user preferences, cached API responses, and simple objects. As soon as you need relational queries — "give me all orders for this customer where status is pending" — Hive becomes awkward. You end up filtering in Dart rather than in the database, which doesn't scale.

Drift — the one I reach for now

Drift (formerly Moor) is a type-safe, reactive SQLite wrapper for Flutter. You define your schema in Dart, run the code generator, and get compile-time verified queries. Relationships, joins, migrations — it handles all of it with a clean API.

// Type-safe, compile-time checked
final query = select(todos)..where((t) => t.dueDate.isSmallerThan(Variable(today)));
final results = await query.get(); // List<Todo>

Drift also supports reactive streams — your UI automatically updates when the underlying data changes. Combined with BLoC or Riverpod, this creates a clean reactive data layer.

My recommendation

Simple caching or preferences: Hive. Any app with real relational data or complex queries: Drift. Raw SQL control for legacy migration: sqflite. For most production apps I build today, Drift is the default choice.
