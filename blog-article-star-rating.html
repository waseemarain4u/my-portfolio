Building a Custom Interactive Star Rating Widget in Flutter

A step-by-step guide to building a fully interactive, gesture-driven star rating widget from scratch — no packages required.

WA

Waseem Akram

Lead Mobile Developer & Software Engineer

Flutter's widget ecosystem is rich, but there are moments where building from scratch teaches you far more than dropping in a package. The star rating widget is a perfect exercise — it involves custom painting, gesture detection, and state management all in one compact component.

Why build it yourself?

There are plenty of star rating packages on pub.dev. But building your own gives you complete control over the visual design, interaction behaviour, half-star support, animation, and accessibility. More importantly, it forces you to understand Flutter's gesture system and custom painting pipeline — knowledge that pays dividends across many other custom widgets.

The core approach: GestureDetector + CustomPaint

The widget is built around two Flutter primitives. GestureDetector captures horizontal drag gestures to update the rating in real-time as the user drags across the stars. CustomPaint draws the stars precisely, filling them based on the current value.

class StarRating extends StatefulWidget {
  final double initialRating;
  final int starCount;
  final ValueChanged<double> onRatingChanged;

  const StarRating({
    super.key,
    this.initialRating = 0,
    this.starCount = 5,
    required this.onRatingChanged,
  });
}

Keeping the widget stateful internally while exposing the value through a callback keeps the API clean — the parent decides what to do with the rating, and the widget manages its own visual state.

Handling gestures for a smooth experience

The key to a satisfying drag interaction is converting the horizontal position of the user's finger into a rating value. You calculate the rating by dividing the drag offset by the total widget width, then clamping it between 0 and the maximum star count. Rounding to the nearest 0.5 gives you half-star precision.

double _calculateRating(double localX, double totalWidth) {
  final raw = (localX / totalWidth) * widget.starCount;
  // Round to nearest half
  return (raw * 2).round() / 2.0.clamp(0, widget.starCount.toDouble());
}

Wiring this to both onHorizontalDragUpdate and onTapDown means the widget responds correctly to both taps and drags.

Drawing the stars with CustomPainter

The StarPainter class uses a Path to trace a five-pointed star polygon for each position. For filled stars it uses the accent colour; for empty stars it uses a muted stroke. For partially filled stars — the half-star case — it uses a clipRect to paint the filled version up to the fractional point, then the empty version for the remainder.

The result is a smooth, pixel-perfect rating widget that matches your app's design language exactly, responds to gestures naturally, and works on every Flutter platform without any external dependency.

Adding animation

Wrapping each star's scale in an AnimationController driven by the current rating adds a satisfying pop when a star becomes selected. Keep it subtle — a scale from 1.0 to 1.2 and back over 150ms is enough to feel responsive without being distracting.
